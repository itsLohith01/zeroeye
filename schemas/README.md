# Schemas

## Build diagnostic metadata

`build-diagnostic.schema.json` describes the JSON metadata files generated by
`build.py` under `diagnostic/build-*.json`. These metadata files accompany the
encrypted `.logd` diagnostic artifacts that contributors include in pull request
notes.

The schema covers:

- diagnostic artifact paths and chunking metadata
- decryption command metadata
- build module counts
- per-module build status, timing, artifact, and output
- pull request note guidance

Example payloads live in `schemas/examples/`:

- `build-diagnostic-valid.json`
- `build-diagnostic-chunked-valid.json`
- `build-diagnostic-invalid.json`
